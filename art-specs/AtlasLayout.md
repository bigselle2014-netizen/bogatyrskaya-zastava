# AtlasLayout — Схема Sprite Atlas, Фаза 1
**Версия**: 1.0 | **Дата**: Апрель 2026 | **Фаза**: 1 (Недели 1–8)

Схема упаковки спрайтов в Unity Sprite Atlas. 4 атласа для Фазы 1.

---

## Общие настройки Unity Sprite Atlas

Применяются ко всем атласам, если не указано иное:

| Параметр | Значение | Примечание |
|---------|---------|-----------|
| Max Texture Size | 2048 | Максимально допустимый размер |
| Compression | ASTC 6x6 | Единственный формат проекта (Baseline: Redmi A2+) |
| Filter Mode | Bilinear | Баланс качества/производительности |
| Generate MipMaps | OFF | 2D игра с фиксированным масштабом |
| Allow Rotation | OFF | Предотвращает нежелательный поворот тайлов |
| Tight Packing | Различается | Персонажи ON, тайлы OFF |
| Padding | 4px | Отступ между спрайтами в атласе |
| Read/Write Enabled | OFF | Экономия памяти |
| sRGB | ON | Цветовое пространство проекта |

**Именование атласов** в Unity:
```
SA_Towers_Phase1
SA_Enemies_Phase1
SA_Tileset_BerezoviyTrakt
SA_UI_Phase1
```

## Хендовер → Unity Dev Middle

**Что получает Dev**: ZIP с отдельными PNG-файлами (не атласы) + данный документ.

**Шаги настройки атласов в Unity**:

1. **Создать Sprite Atlas**: `Assets → Create → 2D → Sprite Atlas`
2. **Назвать** согласно схеме: `SA_Towers_Phase1` и т.д.
3. **Настройки** (в Inspector атласа):
   - Type: Master
   - Include in Build: ON
   - Allow Rotation: OFF
   - Tight Packing: ON (башни, враги, UI) / OFF (тайлы)
   - Padding: 4
   - Filter Mode: Bilinear
   - Max Texture Size: 2048
   - Format: ASTC 6x6
4. **Добавить спрайты**: перетащить папку с PNG в Objects раздел атласа
5. **Pack Atlas**: нажать кнопку Pack Atlas в Inspector
6. **Проверить** Preview атласа — все спрайты уместились в 2048×2048

**Критерии приёмки атласов**:
- [ ] Ни один атлас не превышает 2048×2048px
- [ ] Compression: ASTC 6×6 установлен на все атласы
- [ ] Generate MipMaps: OFF
- [ ] Allow Rotation: OFF
- [ ] Суммарный размер всех атласов в памяти GPU ≤ 4 MB
- [ ] Нет спрайтов, оставшихся за пределами атласов (orphan sprites)
- [ ] Проверка на Redmi A2+: FPS не падает при загрузке всех 4 атласов

## Атлас 5: SA_Towers_Phase2

**Итоговый размер текстуры**: 2048×2048px
**Tight Packing**: ON
**Содержимое**: 10 башен уровней 2-3 × 3 состояния × анимационные кадры

### Расчёт содержимого

```
10 башен × 3 состояния = 30 spritesheets
Каждый spritesheet разрезается на кадры 256×256px:
  idle: 4 кадра = 4 спрайта
  attack: 6 кадров = 6 спрайтов
  upgrade: 5 кадров = 5 спрайтов
Итого: 10 × (4 + 6 + 5) = 150 спрайтов по 256×256px

2048×2048 при ячейке 256×256:
  8 колонок × 8 строк = 64 ячейки
  Нужно: 150 спрайтов → требуется несколько страниц

РЕШЕНИЕ: Unity Sprite Atlas с Multi-Page.
  Страница 1: 64 спрайта (DR-2, DR-3, VL-2, VL-3 — полностью + часть LU-2)
  Страница 2: 64 спрайта (LU-2 остаток, LU-3, BE-2, BE-3, ZN-2 — полностью + часть ZN-3)
  Страница 3: 22 спрайта (ZN-3 остаток + резерв)

АЛЬТЕРНАТИВА (рекомендуемая): разбить на 3 атласа 2048×2048:
  SA_Towers_Phase2_A (DR-2, DR-3, VL-2, VL-3) = 60 спрайтов
  SA_Towers_Phase2_B (LU-2, LU-3, BE-2, BE-3) = 60 спрайтов
  SA_Towers_Phase2_C (ZN-2, ZN-3) = 30 спрайтов
```

> **Примечание для Dev**: при Tight Packing ON Unity может уместить больше спрайтов. Рекомендуется начать с 1 атласа 2048×2048 и проверить — если не влезает, разбить на 2-3.

## Атлас 6: SA_Enemies_Phase2

**Итоговый размер текстуры**: 1024×512px
**Tight Packing**: ON
**Содержимое**: 8 новых врагов × 2 состояния (run + death) × кадры

### Расчёт содержимого

```
8 врагов × 2 состояния:
  run: 6 кадров × 128×128px = 768×128px spritesheet → 6 спрайтов
  death: 4 кадра × 128×128px = 512×128px spritesheet → 4 спрайта
Итого: 8 × (6 + 4) = 80 спрайтов по 128×128px

1024×512 при ячейке 128×128:
  8 колонок × 4 строки = 32 ячейки
  Нужно: 80 спрайтов → не влезает в 1024×512

РЕШЕНИЕ: увеличить до 2048×1024
  2048×1024 при 128×128:
  16 колонок × 8 строк = 128 ячеек
  Занято: 80 спрайтов
  Свободно: 48 (резерв для модификаторов/дополнительных врагов)
```

## Атлас 9: SA_VFX

**Итоговый размер текстуры**: 1024×512px
**Tight Packing**: ON
**Содержимое**: 7 типов VFX-эффектов

### Расчёт содержимого

```
7 VFX spritesheets разрезаются на кадры:
  vfx_sword_slash: 3 × 128×128 = 3 спрайта
  vfx_fire_burst: 5 × 256×256 = 5 спрайтов (крупные!)
  vfx_arrow_trail: 4 × 128×64 = 4 спрайта
  vfx_arcane_circle: 6 × 256×256 = 6 спрайтов (крупные!)
  vfx_heal_particles: 5 × 128×128 = 5 спрайтов
  vfx_enemy_death: 4 × 128×128 = 4 спрайта
  vfx_resource_pickup: 3 × 64×64 = 3 спрайта

Крупные спрайты (256×256): 11 штук = 11 × 256×256 = 2816×256 при ряде
Мелкие спрайты (128×128 и менее): 19 штук

1024×512 — может не хватить для 11 крупных.

РЕШЕНИЕ: увеличить до 2048×512
  Крупные в верхней строке (2048px / 256px = 8 ячеек, нужно 11 → 2 строки)
  2048×512 при смешанных размерах + Tight Packing вместит всё.

АЛЬТЕРНАТИВА: 1024×1024 — более квадратный, оптимальнее для GPU.
```

### Использование памяти

- Текстура 1024×1024, ASTC 6×6 ≈ **170 KB** (при 1024×1024)
- Или 2048×512, ASTC 6×6 ≈ **170 KB**
- 1 draw call для всех VFX

## Критерии приёмки (Фаза 2, все атласы)

- [ ] Ни один атлас не превышает 2048×2048px
- [ ] Compression: ASTC 6×6 на все атласы и отдельные текстуры
- [ ] Generate MipMaps: OFF
- [ ] Allow Rotation: OFF
- [ ] Tight Packing: ON для персонажей/VFX/UI, OFF для тайлов
- [ ] Суммарный размер на экране ≤ 50 MB GPU (фактически ~3 MB)
- [ ] Draw calls в бою ≤ 8
- [ ] Нет orphan sprites за пределами атласов
- [ ] Все атласы проверены на Redmi A2+ (baseline device)
- [ ] FPS стабильно ≥ 30 при загрузке всех атласов текущего экрана
