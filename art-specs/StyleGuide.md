# Style Guide — Богатырская Застава
**Версия**: 1.0 | **Дата**: Апрель 2026 | **Фаза**: 1 (Недели 1–8)

---

## 1.1 Общий стиль

**Направление**: Мультяшный 2D flat-art с лёгкой объёмностью через cel-shading.  
**Вдохновение**: Plants vs Zombies (читаемость силуэта, крупные формы) + русский лубок (орнамент, яркость, народный юмор).  
**Ориентация персонажей**: 3/4 сверху-сбоку. Камера смотрит с лёгким наклоном сверху (~30°), персонаж виден немного сверху и сбоку. Башни выглядят «объёмно» — видна верхняя плоскость шлема/плеч.

**Что НЕЛЬЗЯ**:
- Реалистичные пропорции тела
- Тёмные, грязные, «гриндарк» цвета у героев
- Аниме-стиль (большие наивные глаза аниме ≠ стиль этого проекта)
- Западная fantasy без русских деталей (кельтские руны, нордические орнаменты — замените русскими)

**Ключевые принципы читаемости**:
- Силуэт башни узнаваем на расстоянии 50 см при диагонали экрана 5"
- Цвет-маркер фракции занимает минимум 30% площади спрайта
- Враги всегда темнее и холоднее героев — контраст очевиден

## 1.3 Правила силуэта

**Основное требование**: каждая башня читается как уникальный силуэт на 5" экране (720p) с расстояния 50 см. Для проверки: уменьши спрайт до 64×64px и закрась его однотонным — должно быть понятно, что за башня.

**Контраст башня/фон**: минимум 4:1 по WCAG (проверяется через https://webaim.org/resources/contrastchecker/). Башни светлые на зелёном фоне биома — обводка `#1A1A1A` 2px обеспечивает разделение.

**Цвет-маркер фракции** занимает не менее 30% площади спрайта 256×256px:
- DR: синий `#2E5FA3` — кольчуга + плащ
- VL: фиолетовый `#6B2FA0` — мантия
- LU: зелёный `#2E7D32` — жилет + штаны
- BE: красный `#C62828` — торс + детали брони
- ZN: белый `#F5F5F5` — сарафан + платок

**Уникальные силуэтные маркеры** (то, что делает башню неповторимой в силуэте):
- DR-1 Ратник: щит справа, выступающий за тело; шлем со шпилем
- VL-1 Ведун: длинный посох выше головы; развевающийся рукав мантии
- LU-1 Стрелец: натянутый лук (горизонтальный выступ); меховая шапка-ушанка
- BE-1 Рьяный: широко расставленные руки; топор на плече; широкие плечи без брони
- ZN-1 Травник: посох с цветами (характерная форма кроны из лепестков); покатые плечи

## 1.4 Пропорции персонажей

**Система измерения**: «голов», где голова = 1 единица. Для 256px спрайта голова ≈ 52–64px.

| Уровень башни | Высота (голов) | Ширина | Детализация |
|---------------|---------------|--------|------------|
| Уровень 1 | 4–4.5 головы | Плечи = 2 головы | Простая, читаемая |
| Уровень 2 | 5 голов | Плечи = 2.2 головы | Детали брони, паулдроны |
| Уровень 3 | 5.5–6 голов | Плечи = 2.5–3 головы | Монументальные, эпические, декоративные |

**Голова**:
- Составляет 1/4 от общей высоты тела (не считая оружие и подставку)
- Ширина головы = высота головы × 0.9 (чуть уже — не круглая, но мягкая)
- Глаза: занимают 1/3 высоты лица, расположены в верхней половине головы
- Размер глаза (один) = ≈14–16px на 256px спрайте

**Руки**:
- Длина руки от плеча до кисти = 1.5 головы
- Кисть = 0.6 головы (заметная, выразительная)
- Пальцы не прорисовываются детально на уровне 1 — варежка-рукавица или кулак

**Ноги**:
- Длина ног = 1.5–2 головы
- Ступни видны только при 3/4-виде (видна одна ступня, другая скрыта)

## 1.5 Cel-shading правило

**Три тональных слоя** для каждого цветового пятна:

| Слой | Правило расчёта | Пример (DR-1 синий `#2E5FA3`) |
|------|-----------------|-------------------------------|
| Основной | Базовый HEX | `#2E5FA3` |
| Тень | Основной –30% brightness (HSB) | `#1A3660` |
| Свет | Основной +20% brightness (HSB) | `#4A7FCC` |

**Направление освещения**: источник сверху-справа (угол ≈ 10–2 часа).
- Свет падает: на верхнюю правую часть форм
- Тень ложится: на нижнюю левую часть форм
- Контур тени: мягкий, размытие 1px в Photoshop допустимо

**Переходы**:
- НЕТ градиентов — только резкая граница между основным и тенью
- Граница тени = прямая линия или простая кривая (не сложные фигуры)
- Блик (свет): маленький белый `#FFFFFF` полуовал на самой выступающей части (шлем, плечо)

**Обводка**:
- Толщина: 2px для спрайтов 256×256px
- Цвет: основной HEX цвет фракции –60% brightness (HSB)
  - DR: `#2E5FA3` → `#112244`
  - VL: `#6B2FA0` → `#2A1240`
  - LU: `#2E7D32` → `#113014`
  - BE: `#C62828` → `#4D0F0F`
  - ZN: `#F5F5F5` → `#606060`
- Внутренние линии деталей (складки, стыки): 1px, цвет тени соответствующей зоны

## 1.6 Состояния спрайтов

Каждая башня имеет **3 ключевых кадра**. Это не анимации — это статичные спрайты, Unity интерполирует или переключает между ними.

### idle — Боевая стойка в покое
- Башня стоит в лёгкой готовности, оружие в руке (не опущено)
- Взгляд в сторону врагов (влево по экрану — врага нет, но башня «смотрит»)
- Лёгкий наклон вперёд (≈5°) — не расслаблена, но и не напряжена
- Это базовый кадр, от которого считаются все остальные

### attack — Пик атаки
- **Один кадр** — момент максимального усилия (не замах, не отдача — именно удар/выстрел)
- Тело наклонено или развёрнуто в сторону атаки
- **Визуальный эффект атаки** обязателен: добавляется как отдельный элемент поверх спрайта
  - Мечник: белая/синяя вспышка `#7FB8FF` радиусом 32–48px у клинка
  - Стрелок: смазанный след стрелы (3 линии `#C8A020`, длина 64px)
  - Маг: свечение навершия посоха `#F0C040`, радиус 40px, opacity 80%
  - Берсерк: оранжевые частицы огня `#FF6D00` вокруг топора, 6–8 частиц
  - Знахарь: зелёное свечение `#80FF80` из ладони, радиус 48px

### upgrade — Момент улучшения
- **Золотое свечение**: радиальный градиент `#FFD700` → прозрачный, центр на торсе башни, радиус 96px
- На спрайте появляется **новая деталь экипировки** (конкретно для каждой башни — см. SpriteSpec)
- Рунические символы: 3–5 символов в стиле кириллической вязи, цвет `#F0C040`, size 16–20px, расположены вокруг башни (летят вверх)
- Старая деталь заменяется или дополняется — должна быть визуальная разница с idle

## 1.7 Враги — стиль

**Общие правила**:
- Враги — существа из русского фольклора (нечисть, нежить, чудища), не аниме-монстры и не western-fantasy
- Цветовая гамма: холодная, тёмная, контрастирует с тёплыми цветами башен
- Основные цвета нечисти: `#3D1A5C` (тёмно-фиолетовый), `#1A3B1A` (болотный), `#4A1A1A` (тёмно-красный)
- Глаза нечисти: светятся `#FF4400` (оранжево-красный) — читаются с любого расстояния

**Размеры**:
- Стандартный враг: 128×128px (50% от башни 256px)
- Мелкая нечисть (быстрые): 96×96px (37.5% от башни)
- Боссы: 256×256px (=башне)

**Анимация бега (6 кадров)**:
- Кадр 1–2: левая нога впереди, тело наклонено вперёд на 10°
- Кадр 3: обе ноги вместе (прыжок между шагами), тело выпрямлено
- Кадр 4–5: правая нога впереди, тело наклонено
- Кадр 6: обе ноги вместе (зеркальный кадр 3)
- Важно: руки движутся противофазно ногам (левая нога = правая рука вперёд)
- Недопустимо: «скользкое» движение без смещения конечностей

**Фракционные отличия врагов** (для будущих биомов):
- Берёзовый тракт: духи, лешие, мертвяки — болотно-зелёные тона `#1A3B1A`
- Кикиморины топи: кикиморы, водяные — сине-зелёные `#1A3B4A`
- Пещеры Кощея: скелеты, личи — кость `#C8B89A` + тьма `#1A1A3E`

## 1.8 UI визуальный язык

**Концепция**: деревянные таблички и рамки с русской резьбой, металлические уголки и заклёпки, пергамент для текста.

**Кнопки**:
- Размер: 256×64px (стандартная), 128×64px (квадратная-иконка)
- Фон: текстура тёмного дерева (базовый цвет `#3A2510`) с рисунком резьбы
- Рамка кнопки: 3px, цвет `#8B6914` (тёмное золото)
- Уголки: металлические заклёпки 8×8px, цвет `#C8860A`
- Текст на кнопке: `#F5F0E8` (тёплый белый), размер ≥14px при 1080p
- Hover-состояние: кнопка светлее +15% brightness
- Нажатое состояние: кнопка темнее –10%, сдвиг текста +2px вниз

**Рамки UI-панелей**:
- Толщина рамки: 4–6px
- Внешняя рамка: `#C8860A` (золото)
- Внутренняя рамка: `#8B6914` (тёмное золото)
- Углы: скруглённые 8px или декоративные (орнамент)
- Фон панели: `#1A1A2E` с opacity 90% или `#2A2A3E`

**Иконки башен** (для UI, не игровые спрайты):
- Размер: 64×64px
- Фон: круг или ромб цвета фракции (основной HEX)
- Поверх: силуэт башни ур.1 в белом `#FFFFFF`
- Рамка иконки: 2px `#C8860A`
- Именование: `icon_DR-1.png`, `icon_VL-1.png` и т.д.

**Шрифт**:
- Стиль: без засечек, поддержка кириллицы (рекомендуемые: Russo One, Exo 2, или Rubik)
- Минимальный размер: 14px при разрешении 1080p, 11px при 720p
- Заголовки: Bold, цвет `#FFD700` или `#F5F0E8`
- Основной текст: Regular, цвет `#F5F0E8`
- Вторичный текст: Light, цвет `#A09880`
- НЕ использовать латиницу в русских текстах (кроме ID башен)

**Числа и ресурсы**:
- Золото (монеты): иконка 24×24px `#FFD700`, число рядом `#FFD700`
- Жизни: сердце 24×24px `#C62828`, число рядом `#F5F0E8`
- Мана/очки: кристалл 24×24px `#6B2FA0`, число рядом `#D0A0FF`

## 1.9 Технические ограничения для художника

### Размеры холстов (только степени двойки)

| Тип ассета | Размер холста | Примечание |
|-----------|--------------|-----------|
| Башни (все уровни) | 256×256px | Персонаж занимает ~80% холста |
| Стандартный враг | 128×128px | Персонаж занимает ~85% холста |
| Мелкая нечисть | 128×128px (с полями) или 96×96px | Уточнить с Dev — 96 не степень двойки; рекомендуется 128×128 |
| Тайлы биома | 128×128px | Заполнение до края, без полей |
| Иконки башен (UI) | 64×64px | |
| Кнопки UI | 256×64px | |
| Parallax фон | 1920×1080px | Исключение из правила — UI-слой |

> **Важно**: 96×96px — НЕ степень двойки. Согласовано с Lead Dev использовать 128×128px с полем для мелкой нечисти.

### Форматы файлов

| Требование | Значение |
|-----------|---------|
| Формат | PNG-24 (RGB + Alpha channel) |
| Цветовое пространство | sRGB |
| Глубина цвета | 8 бит на канал (24-bit + 8-bit alpha) |
| Прозрачность | Полная (не premultiplied alpha) |
| Сжатие PNG | Максимальное (уровень 9) — Unity сам конвертирует в ASTC |

### Именование файлов

**Башни**:
```
{TowerID}_{state}.png
Примеры:
  DR-1_idle.png
  DR-1_attack.png
  DR-1_upgrade.png
  VL-1_idle.png
```

**Враги**:
```
{EnemyID}_{state}.png
Примеры:
  E_PESHIY_idle.png
  E_PESHIY_run.png
  E_BRONEY_idle.png
  E_BYSTR_run.png
  E_LETUN_idle.png
```

**Тайлы**:
```
tile_{biome}_{category}_{variant}.png
Примеры:
  tile_bereza_ground_01.png
  tile_bereza_path_straight.png
  tile_bereza_deco_birch_01.png
```

**UI**:
```
ui_{element}_{state}.png
Примеры:
  ui_btn_play_normal.png
  ui_btn_play_pressed.png
  ui_icon_DR-1.png
  ui_progressbar_hp.png
```

### Правила передачи файлов

- Передавать **отдельными PNG** — Unity Sprite Editor нарезает атласы самостоятельно
- НЕ использовать TexturePacker `.tpsheet` — не требуется
- Структура папок при передаче:
```
art-delivery/
  towers/
    DR-1_idle.png
    DR-1_attack.png
    DR-1_upgrade.png
    [остальные башни...]
  enemies/
    E_PESHIY_idle.png
    [...]
  tiles/
    tile_bereza_ground_01.png
    [...]
  ui/
    ui_btn_play_normal.png
    [...]
  parallax/
    bg_bereza_layer1.png
    bg_bereza_layer2.png
```

## Хендовер → Unity Dev Middle

**Формат передачи**: ZIP-архив `art-phase1-week{N}.zip` с папочной структурой выше.

**Чеклист перед передачей**:
- [ ] Все PNG имеют Alpha-канал (не JPG, не RGB без alpha)
- [ ] Размеры кратны степени двойки (32/64/128/256/512)
- [ ] Именование файлов точно соответствует спецификации (проверить регистр)
- [ ] Нет слоёв Photoshop — сохранено как flatten PNG
- [ ] Цветовое пространство: sRGB (не AdobeRGB, не P3)
- [ ] Размер файла: башня 256×256 ≤ 150KB, тайл 128×128 ≤ 50KB

1. Импортирует PNG в Unity → `Assets/Art/Sprites/`

4. Проверяет на Redmi A2+ (Baseline device)
