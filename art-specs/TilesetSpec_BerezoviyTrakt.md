# TilesetSpec — Берёзовый Тракт
**Версия**: 1.0 | **Дата**: Апрель 2026 | **Фаза**: 1 (Недели 1–8)

Полная спецификация тайлового сета «Берёзовый тракт» для Unity 2D Tilemap. Биом уровней 1–17.

---

## Технические параметры

| Параметр | Значение |
|---------|---------|
| Размер тайла | 128×128px |
| Формат | PNG-24 с Alpha-каналом |
| Цветовое пространство | sRGB |
| Совместимость | Unity 2D Tilemap, Rule Tile, Sprite Atlas |
| Паттерн именования | `tile_bereza_{category}_{variant}.png` |
| Максимальный атлас | 2048×512px (16 тайлов в ряд) |
| Передача в Unity | Отдельные PNG → Unity Sprite Atlas |

**Палитра биома** (все тайлы используют только эти цвета):

| Название | HEX | Применение |
|---------|-----|-----------|
| Небо/трава (основной) | `#A8D5A2` | Травяное покрытие тайлов |
| Акцент (цветы/свет) | `#F5E56B` | Ромашки, солнечные блики |
| Земля (тень) | `#6B5A3E` | Грунт пути, тени |
| Свет (листва, роса) | `#D4EDDA` | Блики на траве, светлые листья |
| Кора берёзы | `#F0EFE8` | Белая кора берёзы |
| Пятна берёзы | `#2A2A2A` | Чёрные пятна на коре |
| Листья берёзы | `#A0C060` | Листва берёзы |
| Светлый путь | `#9A8060` | Верхний слой грунтовой дороги |
| Тёмный путь | `#6B5A3E` | Колеи, грязь на пути |
| Дерево/брёвна | `#795548` | Брёвна плетня, колодца |
| Солома | `#D4A020` | Стог сена |
| Небо дальнее | `#C8E6D8` | Небо на горизонте parallax |

## Тайлы земли (4 варианта)

### tile_bereza_ground_01
**Имя файла**: `tile_bereza_ground_01.png`  
**Описание**: Густая трава с ромашками. Основной тайл поля. Трава разной высоты (8–14px): мелкие пучки, 3–4 лезвия в пучке. Ромашки: 3 штуки, белые лепестки `#FFFFFF` + жёлтое сердце `#F5E56B`, высота 16–20px. Трава `#A8D5A2`, стебли `#6B8A4A`, земля под травой `#8A7A5A`.  
**Правила тайлинга**: стыкуется со всеми ground-тайлами без разрывов. Стыки со смежными тайлами сглажены (нет чёткой границы). Тайл асимметричен (ромашки в разных местах) — при повторении не создаёт явного паттерна.  
**Зоны прозрачности (alpha)**: нет — полностью залит (используется как базовый слой Tilemap).  
**Цветовые ограничения**: строго в палитре биома; ромашки — только `#FFFFFF` + `#F5E56B`.

---

### tile_bereza_ground_02
**Имя файла**: `tile_bereza_ground_02.png`  
**Описание**: Трава с васильками. Аналог ground_01, но вместо ромашек — 2–3 василька. Лепестки василька: `#4A90C8` (синий), сердце `#1A3A5C`. Трава та же. Используется для разнообразия поля — чередовать с ground_01 в соотношении 2:1.  
**Правила тайлинга**: стыкуется со всеми ground-тайлами. Совместим с ground_01 как сосед.  
**Зоны прозрачности**: нет.  
**Цветовые ограничения**: василёк — единственное отступление от основной палитры (синий); остальное строго в палитре биома.

---

### tile_bereza_ground_03
**Имя файла**: `tile_bereza_ground_03.png`  
**Описание**: Примятая трава. Используется вплотную к пути (`_path_*`). Трава приплюснута и короче (4–8px), направление — к центру тайла (трава «примята» проходящими). Цветов нет. Трава бледнее: `#8AC888` (не такая яркая). Лёгкие следы (2 параллельные вмятины 4px `#6B5A3E`).  
**Правила тайлинга**: специально для соседства с path-тайлами. С одной стороны (левой или правой, верхней или нижней) — переход к dirt-текстуре. С других сторон — стыкуется с ground_01/ground_02.  
**Зоны прозрачности**: нет.  
**Цветовые ограничения**: только в палитре биома.

---

### tile_bereza_ground_edge
**Имя файла**: `tile_bereza_ground_edge.png`  
**Описание**: Переходный тайл трава→земля/обрыв. Одна половина тайла — трава (ground_01-стиль), другая — обнажённая земля `#6B5A3E` с камешками (`#8A7A5A` и `#4A3A2A`, размер 4–8px). Переход мягкий, не резкий (3–4px плавный gradient-like переход). Используется по краям игрового поля.  
**Правила тайлинга**: не стыкуется с собой по оси перехода — только по краям. Существует в 4 версиях поворота (Unity Rule Tile настроит ротацию).  
**Зоны прозрачности**: нет.

## Тайлы пути врагов (4 варианта)

### tile_bereza_path_straight
**Имя файла**: `tile_bereza_path_straight.png`  
**Описание**: Прямой участок грунтовой дороги, горизонтальный. Ширина дороги: 80px из 128px холста (по 24px травы с каждой стороны). Грунт: `#9A8060` основа, колеи `#6B5A3E` (2 параллельные колеи, 6px каждая, отступ от края 20 и 54px). Лужицы: 1–2 небольшие лужи `#6A7A8A` (стальной серо-голубой), размер 16×8px. Следы копыт: `#5A4A2E`, дугообразные отпечатки, 8–10px.  
**Правила тайлинга**: стыкуется горизонтально с path_straight, path_curve, path_cross. По верхней и нижней стороне: 24px полосы травы совпадают с ground_03 или ground_01.  
**Зоны прозрачности**: нет.  
**Цветовые ограничения**: путь — только `#9A8060` + `#6B5A3E` + `#6A7A8A` для луж.

---

### tile_bereza_path_curve_l
**Имя файла**: `tile_bereza_path_curve_l.png`  
**Описание**: Поворот грунтовой дороги влево (дорога входит справа, выходит вверх). Дуга пути: плавная четверть окружности. Колеи по дуге. Внутренний угол: трава с чуть более высокой растительностью (нарастает в уголке). Внешний угол: лужа или отдельный камень.  
**Правила тайлинга**: правая сторона — совместима с path_straight (вход пути); верхняя — совместима с path_straight (выход пути); левая и нижняя — трава ground_03.  
**Зоны прозрачности**: нет.

---

### tile_bereza_path_curve_r
**Имя файла**: `tile_bereza_path_curve_r.png`  
**Описание**: Поворот вправо (дорога входит слева, выходит вверх). Зеркальный вариант path_curve_l.  
**Правила тайлинга**: левая сторона — вход пути; верхняя — выход пути; правая и нижняя — трава.  
**Зоны прозрачности**: нет.

---

### tile_bereza_path_cross
**Имя файла**: `tile_bereza_path_cross.png`  
**Описание**: Перекрёсток/развилка. Дорога проходит горизонтально и вертикально — крест. Центр: грязная, наиболее примятая земля `#6B5A3E`, много луж и следов. Используется для усложнённых карт или как декоративный элемент (развилка).  
**Правила тайлинга**: все 4 стороны совместимы с path_straight и path_curve.  
**Зоны прозрачности**: нет.

## Тайлы декора (8 вариантов)

> **Важно**: декор-тайлы накладываются **поверх** ground- и path-тайлов. Нижняя часть холста — прозрачная (alpha), верхняя/центральная — объект декора.

### tile_bereza_deco_birch_01
**Имя файла**: `tile_bereza_deco_birch_01.png`  
**Описание**: Молодая тонкая берёза. Ствол 8px толщиной, высота ствола ~96px (занимает большую часть холста вертикально). Кора белая `#F0EFE8` с чёрными пятнами `#2A2A2A` (овальные, 8–14px, нерегулярно). Крона: неплотная, ~48px в диаметре, состоит из 12–16 мелких листиков (треугольники/капли) `#A0C060`, `#8AB050`. Листья слегка просвечивают небо (не сплошная масса). Тень ствола: 4px серая линия `#8A8A8A` справа.  
**Правила тайлинга**: не стыкуется (независимый объект). Нижний центр тайла = «точка посадки» дерева (совпадает с поверхностью земли под ним). Размещать на ground_01/02.  
**Зоны прозрачности (alpha)**: вся нижняя половина тайла прозрачна. Крона — полупрозрачные промежутки между листьями (alpha).  
**Цветовые ограничения**: кора `#F0EFE8` + `#2A2A2A`; листья в диапазоне `#80B040`–`#C0D880`.

## Parallax-слои

Два фоновых слоя, каждый **1920×1080px**, PNG-24.

---

### Parallax Layer 1 — Задний план
**Имя файла**: `bg_bereza_layer1.png`  
**Скорость parallax**: ×0.3 (почти не движется)  
**Передача в Unity**: `Assets/Art/Backgrounds/`

**Что нарисовано**:
- **Небо**: верхние 50% холста (540px). Плавный градиент `#C8E6D8` (горизонт, светло-зелёный) → `#A8D5B8` (зенит). Небо пустое, без облаков — чистый летний день. Опционально: 1–2 перистых облака `#FFFFFF` с opacity 60%, очень тонкие и высокие.
- **Горизонт**: линия горизонта на высоте 55% от дна (594px от верха). Мягкая, нечёткая — деревья размытые (blur 2px).
- **Далёкие берёзы**: 8–10 берёз высотой 160–200px, расположены вдоль горизонта. Стволы `#F0EFE8` (белые полоски шириной 6–8px), кроны `#8AB050` с opacity 70% (дымка расстояния). Всё слегка desaturated (+10% туман): `#A0B898`.
- **Поле вдали**: полоса 60–80px высотой у горизонта. Жёлто-зелёное поле `#C8D870`, создаёт глубину.

**Ощущение**: лёгкость, простор, летний день. Мирная деревенская Россия.

---

### Parallax Layer 2 — Средний план
**Имя файла**: `bg_bereza_layer2.png`  
**Скорость parallax**: ×0.6  
**Передача в Unity**: `Assets/Art/Backgrounds/`

**Что нарисовано**:
- **Деревья среднего плана**: 5–7 берёз высотой 300–400px, расположены группами (роща слева, одиночные справа). Детализованнее слоя 1 — видны чёрные пятна на коре, листья крупнее. Стволы `#F0EFE8`, кроны `#A0C060` (насыщеннее, чем задний план).
- **Деревенька вдали**: 3–4 дома на среднем плане (справа от центра). Дома — силуэты: прямоугольники `#8A7A5A` (стены) + треугольники `#795548` (крыши). Размер домов: 80–120×60–80px. Из одной трубы — светло-серый дымок `#D0D0D0`.
- **Плетень**: горизонтальная линия плетня на высоте 20% от дна (216px от верха), проходит по всей ширине с небольшими разрывами. Высота плетня 32px, цвет `#795548`.
- **Трава переднего слоя**: полоса высокой травы (32–48px) у нижнего края слоя. Трава `#A8D5A2` с тёмными стеблями `#4A7A30`. Несколько крупных ромашек `#F5E56B`.

**Ощущение**: деревенская жизнь на расстоянии, тепло и спокойствие. Деревья дают ощущение глубины.

## Хендовер → Unity Dev Middle

**Что передаётся**: 16 тайлов (128×128px PNG) + 2 parallax-слоя (1920×1080px PNG).

**Список тайлов**:
```
tile_bereza_ground_01.png    tile_bereza_ground_02.png
tile_bereza_ground_03.png    tile_bereza_ground_edge.png
tile_bereza_path_straight.png  tile_bereza_path_curve_l.png
tile_bereza_path_curve_r.png   tile_bereza_path_cross.png
tile_bereza_deco_birch_01.png  tile_bereza_deco_birch_02.png
tile_bereza_deco_fence.png     tile_bereza_deco_stone_01.png
tile_bereza_deco_stone_02.png  tile_bereza_deco_flower.png
tile_bereza_deco_well.png      tile_bereza_deco_hay.png
```

**Список фонов**:
```
bg_bereza_layer1.png    bg_bereza_layer2.png
```

**Формат передачи**: папки `tiles/` и `backgrounds/` в ZIP.

**Что Dev делает**:
1. Тайлы → `Assets/Art/Tiles/BerezoviyTrakt/`
2. Создать Unity Tile Palette, расположить тайлы
3. Упаковать в `SA_Tileset_BerezoviyTrakt`
4. Фоны → `Assets/Art/Backgrounds/`, Layer Order –10 и –5
5. Parallax-компонент: Layer1 speed 0.3, Layer2 speed 0.6
6. Ground-тайлы: Allow Rotation OFF, Tight Packing OFF
7. Deco-тайлы: Allow Rotation OFF (деревья не поворачивать), возможен flipX

**Критерии приёмки**:
- [ ] Все ground-тайлы стыкуются без видимых швов
- [ ] Path-тайлы создают непрерывную дорогу при любом сочетании прямых + поворотов
- [ ] Deco-тайлы не создают артефактов на краях (alpha корректный)
- [ ] Parallax-слои при движении камеры не имеют пустых краёв (1920px достаточно)
- [ ] Цветовая гамма всех тайлов в рамках палитры биома
