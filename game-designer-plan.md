# Game Designer — Личный план работы
## Проект: "Богатырская Застава" (TD + Idle + Roguelite, рынок РФ)

---

## 1. Фаза 1 — Прототип (Месяц 1–2, ~$30K)

### Критерий выхода из фазы:
**D1 self-report ≥ 70%** тестеров отвечают "хочу ещё сыграть" после первой сессии.
Метод: Google Forms, выборка ≥ 20 человек, период измерения — последние 3 дня.

---

### Что делаю конкретно:

**Неделя 1–2: Ядро механики на бумаге**
- Пишу Core Mechanics Document (CMD): описываю все 5 фракций, их базовые характеристики, механику размещения
- Использую только канонические имена башен из game-bible.md (секция 1): Ратник, Витязь, Святогор; Ведун, Чародей, Мерлин Русич; Стрелец, Соколиный глаз, Добрыня-стрелок; Рьяный, Буян, Еруслан; Травник, Знахарка, Белояр
- Составляю таблицу синергий v0.1 — минимум 4 рабочих комбо из game-bible.md секция 3:
  - S-01 Дружинный строй: 3+ башни Дружины → +30% physical damage
  - S-07 Огненный шар: Берсерк + Волхв рядом (3 клетки) → AOE шар каждые 8 сек, урон ×2.5
  - S-08 Ядовитое исцеление: Лучник + Знахарь рядом → каждый тик яда = +3 HP башне
  - S-06 Богатырская стена: 5 башен Дружины → 10 сек неуязвимости, кулдаун 45 сек
- Определяю базовые числа: HP башен, урон, скорость атаки, стоимость

**Хендовер → к неделе 2:**
- TowerDesignSheet.pdf (фракции, уровни, ключевые черты) → 2D Художник
- Художник начинает концепты башен Дружины → неделя 3

**Неделя 2–3: 10 тестовых уровней**
- Разрабатываю Level Design Document (LDD) на 10 уровней прототипа
- Уровни привязаны к биомам game-bible.md секция 2:
  - Уровни 1–10 — Акт 1: **Берёзовый тракт** (уровни 1–17 канон) и начало **Сёл у реки** (уровни 18–30)
- Волны 1–3: один тип врагов, без механических сюрпризов
- Волны 4–7: 2–3 типа врагов, начало учёбы тактики
- Волны 8–10: синергии становятся необходимы для прохождения

**Хендовер → к неделе 3:**
- BiomeVisualGuide.md (8 биомов, тональность) → 2D Художник
- Художник начинает концепты биомов → неделя 4

**Неделя 3–4: Idle-механизм v0**
- Описываю офлайн-прогрессию: что накапливается (монеты, рунные камни), cap = 8 часов
- Пишу Idle Balance Sheet v0.1: ставки добычи, зависимость от уровня Заставы

**Хендовер → к неделе 3:**
- TowerConfig.json (все 15 башен, stats v1.0) → Unity Dev Middle
- Unity Dev начинает Tower Placement System → неделя 4

**Хендовер → к неделе 4:**
- TutorialFlow.md (пошаговый сценарий + условия skip) → Unity Dev Middle
- Unity Dev начинает Tutorial.cs → неделя 5

**Неделя 5–6: Плейтест-сессии**
- Провожу 2 раунда плейтестов с 10 тестерами каждый
- Готовлю опросник: "хочу ещё сыграть?" (бинарно), что понравилось, где застрял
- Фиксирую все проблемы в Playtest Log, приоритезирую по частоте

**Неделя 7–8: Итерация и финальный плейтест**
- Правлю числа (баланс волн, стоимость башен, скорость прогрессии)
- Финальный плейтест: ≥ 20 тестеров → **критерий выхода: D1 self-report ≥ 70%**

## 2. Фаза 2 — MVP (Месяц 3–5, ~$80K)

### Критерии выхода из фазы (все три одновременно):
- **D1 Retention ≥ 35%**
- **D7 Retention ≥ 12%**
- **IAP conversion ≥ 3%**
Период измерения — последние 3 дня, не пиковые значения.

---

### Что делаю конкретно:

**Месяц 3: Контент 50 уровней + полные 5 фракций**
- Расширяю LDD до 50 уровней по структуре биомов game-bible.md секция 2:
  - **Акт 1:** Берёзовый тракт (1–17) + Сёла у реки (18–30)
  - **Акт 2:** Дикое поле (31–38) + Кикиморины топи (39–43) + Пещеры Кощея (44–50)
  - (Акт 3 — Полночный предел, Чёрная крепость, Ирий-сад — уровни 51–70 — за пределами MVP)
  - Endless Mode (71–100): смешение всех биомов с нарастающей тьмой — за пределами MVP
- Для каждого уровня: кол-во волн, состав, HP/скорость врагов, награды, рекомендуемый состав башен
- Все 15 башен по каноническим именам (game-bible.md секция 1):
  - **Дружина**: Ратник (DR-1), Витязь (DR-2), Святогор (DR-3)
  - **Волхвы**: Ведун (VL-1), Чародей (VL-2), Мерлин Русич (VL-3)
  - **Лучники**: Стрелец (LU-1), Соколиный глаз (LU-2), Добрыня-стрелок (LU-3)
  - **Берсерки**: Рьяный (BE-1), Буян (BE-2), Еруслан (BE-3)
  - **Знахари**: Травник (ZN-1), Знахарка (ZN-2), Белояр (ZN-3)

**Месяц 3: Синергии v1.0 — все 12 из game-bible.md секция 3**

Пишу Synergy Design Document для всех 12 синергий (S-01 — S-12):

Одиночные:
- S-01 Дружинный строй, S-02 Волховской круг, S-03 Лучниковый залп
- S-04 Ярость берсерков, S-05 Знахарский собор, S-06 Богатырская стена

Комбинированные:
- S-07 Огненный шар, S-08 Ядовитое исцеление, S-09 Ледяной щит
- S-10 Кровавый ливень, S-11 Витязь-защитник, S-12 Пепел и лёд

Для каждой: условие активации, эффект, стак/не стак, визуальный сигнал.
Тестирую: нет ли доминирующей стратегии.

**Хендовер → к неделе 6:**
- SynergyConfig.json (12 синергий, условия + числа) → Unity Dev Middle
- Unity Dev начинает SynergySystem.cs → неделя 7

**Месяц 3–4: Roguelite-режим**
- Пишу Roguelite Mode Design Document:
  - Структура рана: 5 этапов × 3–5 мин каждый = 15–25 мин суммарно
  - Random events между этапами: выбор из 3 вариантов (усиление, ресурсы, особая башня)
  - Перманентный прогресс: Рунные Очки → дерево улучшений Заставы (10–15 нод)
  - Разблокировка новых богатырей через раны
  - Метапрогрессия не ломает баланс F2P

**Месяц 4: Монетизационный дизайн**
- Пишу Monetization Design Document:
  - Стартовый пак: 69₽ (1 редкий скин + Рунные камни + Монеты)
  - Battle Pass: 399₽/мес, 30 наград, XP × 1.3, скин — без боевых преимуществ (согласно core loop game-bible.md секция 7)
  - IAP пакеты: 4 варианта (79₽, 199₽, 499₽, 1499₽)
  - Rewarded Video: точки показа (после победы, при поражении, ускорение idle), лимит 1/10–15 мин
  - IAP не показывать до дня 3
- Прописываю onboarding flow день 1–7

**Хендовер → к неделе 8:**
- EventCalendar.json (ивенты до месяца 12) → Lead Dev
- Lead Dev начинает EventSystem.cs → неделя 9

**Месяц 4–5: Idle-прогрессия v1.0**
- Финальная Idle Balance Sheet:
  - База добычи: монеты/час при Заставе ур.1
  - Формула: согласована с Lead Dev (IdleOfflineConfig, неделя 6–7)
  - Макс. офлайн накопление: 8 часов
  - Апгрейд Заставы: 5 уровней

**Хендовер → к неделе 10:**
- PushSchedule.md (сегменты, тексты, триггеры) → Lead Dev
- Lead Dev интегрирует в AppMetrica → неделя 11

**Месяц 5: Плейтесты MVP + балансировка**
- 3 раунда плейтестов: **критерий выхода: D1 ≥ 35%, D7 ≥ 12%, IAP ≥ 3% — все три одновременно**
- Инструмент: Google Sheets с воронкой (install → tutorial_complete → level_1 → day7_return)
- Итерирую баланс если D7 < 12% — усиливаю Idle-награды и события дня 4–7

## 7. Риски в моей зоне ответственности

| Риск | Вероятность | Последствие | Митигация |
|------|------------|-------------|-----------|
| **Одна из 12 синергий становится единственной winning стратегией** | Высокая | Потеря глубины, скука после 5–10 рунов | Тест каждой синергии изолированно + в комбо; нерфить доминанта до релиза; числа менять только с GD-апрув + обновлением game-bible |
| **Баланс уровней: завал в Кикиморины топи / Пещеры Кощея** | Средняя | D7 падает, игроки бросают | Промежуточные плейтесты на каждом акте; "difficulty tuning" сессии отдельно |
| **Idle-прогрессия слишком быстрая → нет смысла возвращаться** | Средняя | D7 < 12% | cap 8 ч = достаточно для 1 апгрейда башни, не больше |
| **Idle-прогрессия слишком медленная → нет удовлетворения** | Средняя | D1 уходят разочарованными | Первые награды ощутимыми: монеты за 1 час при старте |
| **Монетизация ощущается P2W** | Средняя | Негативные отзывы в RuStore | Battle Pass = ×1.3 XP, не сила; все богатыри бесплатны (core loop, секция 7) |
| **Roguelite ран слишком длинный для мобайла** | Средняя | Потеря в середине | Жёстко держать 15–25 мин, кнопка "сохранить ран" на любом этапе |
| **Контент 50 уровней кончается к дню 30** | Низкая | Нет причины играть | Endless Mode (71–100) + Roguelite = infinite loop; ивент-план каждый месяц |
| **Хендовер задержан → Dev не начинает вовремя** | Средняя | Сдвиг всего расписания | Готовить артефакты с буфером 2–3 дня до дедлайна хендовера |
| **Культурный ивент не совпадает по срокам с релизом** | Низкая | Упущен traffic-пик | EventCalendar.json с датами + буфер 2 недели на реализацию |
